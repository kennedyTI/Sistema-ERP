# Sistema ERP v2.3.0 - Máquinas e Status Operacional

## Objetivo

Consolidar a primeira entrega operacional do módulo Impressoras, separando o
cadastro de máquinas da consulta de status e oferecendo uma experiência
responsiva, segura e orientada por permissões.

## Principais entregas

- CRUD de máquinas com ativação, inativação, auditoria e controle de
  concorrência;
- central de status somente leitura, exibindo apenas máquinas ativas;
- modelos de impressora e status administrados de forma segura pelo Django
  Admin;
- permissões granulares integradas ao Django Auth e ao frontend;
- cards de resumo, tabelas configuráveis e colunas reordenáveis por arraste;
- modais responsivos com imagens dos modelos e fallback visual;
- APIs v2 para máquinas, detalhes, resumo e status operacional;
- interface validada em desktop, notebook, tablet e celular.

## Validações executadas

- `python -m compileall -q backend`;
- `pytest -q`: 70 testes aprovados;
- `python manage.py check`: nenhuma inconsistência;
- `npm install`, `npm audit` e `npm run build`: build aprovado e zero
  vulnerabilidades;
- Docker Compose: serviços ativos e migrations finalizadas com exit code 0;
- HTTPS local: login, início, Admin, autenticação, máquinas, status e papel;
- verificação de que máquinas inativas não aparecem no status operacional;
- verificação de ausência de dados locais sensíveis e certificados rastreados.

## Decisões técnicas

- cadastro Ativa/Inativa permanece separado do status operacional;
- status e logs operacionais são somente leitura na API pública e no Django
  Admin;
- preferências de colunas ficam persistidas por usuário no navegador;
- imagens usam `url_imagem` como fonte oficial, sem inferência de caminhos pelo
  frontend;
- Redis, Celery, SNMP e monitoramento automático permanecem fora do escopo.

## Limitações não bloqueantes

- os dados de status ainda dependem de alimentação controlada, sem coleta
  automática;
- o card de substituição de toner usa temporariamente a mensagem de alerta;
- o cadastro ainda recebe fabricante e modelo em texto, sem endpoint dedicado
  de catálogo;
- o build informa apenas o aviso conhecido de chunk JavaScript acima de 500 kB.
